One-line goal: scaffold the Android Capacitor platform (dependency + landscape-lock script + GitHub Actions debug-APK build) so every push to main produces a downloadable unsigned APK — everything short of the Sivert-only signed Play publish.

## Goal

### Step 3 — scripts/android-landscape.mjs (NEW)
Create the file with this exact content:
```js
#!/usr/bin/env node
// scripts/android-landscape.mjs
//
// Patches android/app/src/main/AndroidManifest.xml AFTER `npx cap add android`
// has generated the Android project (in CI — the android/ folder is never
// committed; see .gitignore). Mirrors scripts/ios-landscape.mjs.
//
// What it does:
//   Sets android:screenOrientation="sensorLandscape" on the .MainActivity
//   <activity> element — both landscape rotations allowed, portrait never,
//   regardless of the device's rotation-lock setting. This is the Android
//   equivalent of the iOS UISupportedInterfaceOrientations landscape-only
//   patch.
//
// Idempotent: re-running when the attribute is already correct produces no
// diff and does not rewrite the file. If the attribute exists with a
// different value, it is replaced. Does NOT assume attribute order or the
// exact android:configChanges list Capacitor generates.

import { readFileSync, writeFileSync, existsSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const ROOT = join(__dirname, '..');
const MANIFEST_PATH = join(ROOT, 'android', 'app', 'src', 'main', 'AndroidManifest.xml');

function log(msg) {
  console.log(`[android-landscape] ${msg}`);
}

if (!existsSync(MANIFEST_PATH)) {
  console.error(
    `[android-landscape] ERROR: ${MANIFEST_PATH} does not exist.\n` +
    `This script must run AFTER "npx cap add android" has generated the\n` +
    `Android project (CI step in .github/workflows/android-debug.yml).`
  );
  process.exit(1);
}

let xml = readFileSync(MANIFEST_PATH, 'utf8');
const original = xml;

// Find every <activity ...> opening tag, pick the one for .MainActivity.
// Attributes may span multiple lines; never assume order.
const activityTagRe = /<activity\b[\s\S]*?>/g;
const tags = xml.match(activityTagRe) || [];
const mainTag = tags.find((t) => t.includes('android:name=".MainActivity"'));

if (!mainTag) {
  console.error('[android-landscape] ERROR: no <activity> with android:name=".MainActivity" found in AndroidManifest.xml.');
  process.exit(1);
}

const ORIENTATION = 'sensorLandscape';
let newTag;
if (/android:screenOrientation="[^"]*"/.test(mainTag)) {
  newTag = mainTag.replace(
    /android:screenOrientation="[^"]*"/,
    `android:screenOrientation="${ORIENTATION}"`
  );
} else {
  newTag = mainTag.replace(
    /^<activity\b/,
    `<activity\n            android:screenOrientation="${ORIENTATION}"`
  );
}

xml = xml.replace(mainTag, newTag);
log(`set .MainActivity android:screenOrientation -> ${ORIENTATION}`);

if (xml === original) {
  log('no changes needed (already patched) — manifest untouched on disk.');
} else {
  writeFileSync(MANIFEST_PATH, xml, 'utf8');
  log(`wrote ${MANIFEST_PATH}`);
}

log('done.');
```

```yaml
# Android debug APK for StrikeArc (svingbue).
#
# Scope: UNSIGNED debug build only — proves the Android Capacitor platform
# assembles and packages the web app. Keystore signing + Play Console
# publish are deliberately out of scope (Sivert-only external steps).

  push:
    branches: [main]
    paths-ignore:
      - '**.md'
      - 'docs/**'
      - 'tools/**'
      - 'codemagic.yaml'
  workflow_dispatch:

concurrency:
  group: android-debug-${{ github.ref }}
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      # Capacitor 7's Android template (AGP 8.7.x / Gradle 8.11) targets Java 21.
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 21

      - name: Install npm dependencies
        run: npm ci

      - name: Assemble www/ from web source (denylist mocks/tooling)
        run: npm run copy-web

      - name: Add the Android platform (never committed — generated fresh)
        run: |
          if [ ! -d "android" ]; then
            npx cap add android
          fi

      - name: Sync web assets + plugins into the Android project
        run: npx cap sync android

      - name: Patch AndroidManifest.xml for landscape-only
        run: node scripts/android-landscape.mjs

      - name: Generate app icons/splash from resources/
        # --assetPath resources is REQUIRED here: the repo root also has an
        # assets/ dir (web images), which is @capacitor/assets' default
        # source path — without the flag it would look in the wrong folder.
        run: npx capacitor-assets generate --android --assetPath resources --iconBackgroundColor "#0A0E12"

      - name: Build debug APK
        run: |
          cd android
          chmod +x gradlew
          ./gradlew assembleDebug --no-daemon

      - uses: actions/upload-artifact@v4
        with:
          name: strikearc-android-debug
          path: android/app/build/outputs/apk/debug/app-debug.apk
          if-no-files-found: error
          retention-days: 14
```

## Edge cases a weaker model would miss
1. **`--assetPath resources` is mandatory.** `@capacitor/assets` defaults to an `assets/` source folder — and this repo root HAS one (`assets/`, the web-image dir in copy-web's `ALLOWED_DIRS`, copy-web.mjs:32). Without the flag the generator reads the wrong directory. Note: `codemagic.yaml:98` omits the flag on the iOS side — do NOT "fix" codemagic.yaml (out of scope, file must stay untouched); just flag the observation in your final report.
2. **`[skip ci]` skips GitHub Actions too**, not just Codemagic — that's why the workflow has `workflow_dispatch` and why Option A pairs `[skip ci]` with `gh workflow run`. `workflow_dispatch` is unaffected by `[skip ci]` but only works after the workflow file exists on `main`.
3. **`npm ci` in CI fails on a stale lock file.** Adding `@capacitor/android` to package.json by hand without running `npm install` locally breaks the very first CI step. Step 1 uses `npm install` precisely so `package-lock.json` is regenerated.
4. **`npx cap add android` requires `www/` to exist** (`webDir` from capacitor.config.ts:23) — `npm run copy-web` must precede it, locally (Step 4.1) and in CI (workflow order).
5. **Do not assume the manifest's attribute order or `configChanges` content.** Capacitor's generated `<activity>` tag varies between versions; the script matches any `<activity ...>` tag containing `android:name=".MainActivity"` and inserts/replaces only `android:screenOrientation` (see Step 3 regexes). Never string-match the whole template tag.
6. **`cap sync android` does not regenerate AndroidManifest.xml** (only `cap add` writes it; sync copies web assets + updates plugins), but the workflow still patches AFTER sync to mirror the proven iOS ordering in codemagic.yaml:73–80 — keep that order.
7. **copy-web.mjs needs NO edit**: `'android'` is already in `DENYLIST_DIRS` (line 39) and in the `forbiddenInWww` sanity guard (line 126). Verify, don't touch. But this only matters if `android/` exists at copy-web time — in CI it never does (copy-web runs before cap add); locally it might (harmless, denylisted).
8. **OneDrive + node_modules**: never run recursive shell find/grep from the repo root (times out). Delete the locally generated `android/` after verification (Step 4.7) — hundreds of Gradle files syncing to OneDrive is pointless churn.
9. **RevenueCat on Android with `goog_REPLACE_ME`** (sa-iap.js:43) — the Gradle build compiles fine (the plugin's BillingClient needs no key at build time), and at runtime `init()` never throws (sa-iap.js:88–99 wraps configure in try/catch → degrades to `pro=false`). Consequence: the debug APK enforces the 10-shot gate (`isWeb()` is false on native) with a purchase flow that cannot complete. That is EXPECTED at this stage; do not "fix" it.
10. **`www/` is a build artifact** — never hand-edit it; it is wiped and rebuilt by copy-web.mjs (lines 74–79).
11. **CRLF**: git's CRLF warnings on Windows are benign. The `.mjs` and `.yml` files may be written with CRLF locally; both Node and GitHub Actions accept them. `android/gradlew` is generated on the Linux runner itself (LF, but chmod +x is still needed — the workflow does it).
12. **YAML tabs**: a single tab character in android-debug.yml makes GitHub reject the workflow silently (run never appears). Spaces only; lint with `npx --yes yaml-lint` before committing.
13. **`sensorLandscape`, not `landscape`**: plain `landscape` allows only ONE of the two landscape rotations; `sensorLandscape` allows both (matching iOS `UIInterfaceOrientationLandscapeLeft`+`Right`) and overrides the user's rotation-lock toggle.
14. **`@capacitor/screen-orientation` is installed but unused in web code** (only mentioned in NATIVE.md:57) — the lock is manifest/plist-level by design. Do not add runtime `ScreenOrientation.lock()` calls; a bare-module or wrong import there is exactly the class of bug that crashed WKWebView before (vendored, relative imports only — see sa-iap.js:20 and vendor/revenuecat/).
15. **Do not add `android/` sub-path ignores like the ios/ block** — ignore the whole dir (Step 2 rationale in the .gitignore comment).
