# Academy batch improvement loop

Use this prompt for each Academy batch after Batch 0. It is an execution loop,
not permission to weaken an acceptance gate.

> Work only on the assigned branch and the single Academy batch claimed in
> `COORDINATION.md`. Re-read the mutex and working-tree status before every work
> block. Read the batch's exact spec/plan pair, the rollout index, `STATUS.md`
> and `SESSION-HANDOFF.md` before editing. Preserve protected identifiers,
> storage compatibility and all physics engines byte-for-byte.
>
> Build a compact target picture from the accepted specification: learner
> outcome, one dominant action, evidence hierarchy, truth boundaries, mastery
> transaction, voice/caption cues, accessibility and native package parity.
> Use design skills as critique, while repository laws and the accepted spec
> remain authoritative.
>
> Repeat this loop without asking the owner to verify intermediate work:
>
> 1. Prove the next missing behavior with the smallest relevant failing test.
> 2. Implement the smallest coherent change that satisfies the behavior.
> 3. Run focused model, store, browser, accessibility and voice tests.
> 4. Inspect both target portrait sizes, both engines and reduced motion; repair
>    hierarchy, clipping, focus, target size, copy or truth defects found.
> 5. Re-run the focused gates, then the non-duplicative full gate set.
> 6. Continue until there are no critical defects, every category floor and
>    critical check passes, performance and approved pixel baselines pass, and
>    the provenance-blind pairwise comparison beats the previous generation.
>
> Never approve a changed baseline merely to make the gate green. If a required
> licensed asset, physical-device check, independent human judgment or owner
> release decision is unavailable, finish every safe automated item, fail the
> affected gate closed and record the exact external checkpoint. Commit and
> push a clean, reproducible handoff; move the coordination row to Done only
> when the claimed batch work and its truthful evidence are present.

The default final gate set is `npm run claude:ready`, `npm run test:perf`,
`npm run test:visreg`, `npm run copy-web`, `npm run brand:verify`, development
and strict release voice-pack verification, protected-file hashes, packaged
parity, `git diff --check` and the batch's blind comparison/derived verdict.
Equivalent component results may replace a duplicate rerun only when the exact
commands and fresh evidence are recorded; no critical gate may be omitted.
