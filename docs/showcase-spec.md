# THE OBSERVATORY BROADCAST — showcase spec (Fable 5, 2026-07-10)

**Commission:** transform TWO surfaces only — (1) the front page `home-mock.html`, (2) the 3D geometry view `geometry.html` + `geo3d/` — into a Fable capability showcase. Owner picked directions 1+2 (Broadcast + Observatory), fused. Everything else in the app is OUT OF SCOPE.
**The fusion in one line:** *the universe is slow; the strike is violent.* At rest, both surfaces move like orbital mechanics — long gravitational easing, star-trail persistence, telescope glides. At the MOMENT (a swing, a replay), the grammar snaps to live broadcast — whip-pans, speed ramps, typed lower-thirds, a hard freeze.

2. 3. Front-page contracts: interactive at 0ms; `sa.stat.*` data-alive (same keys/staleness as today); destinations Ball Flight / Outcome / Strike Window / Academy / Diagnose (5 now); help affordance; session-gated entry choreography (`sessionStorage`), skippable on any input, ≤2.8s.
4. A11y floor = the 16 guardrails in `docs/home-creative-research.md` §"Non-negotiable guardrails" (reduced-motion parity, no gesture-only affordances, ≥44px moving-target hit areas, hit-testing during motion, one polite live region, `.sa-focus` double ring, landscape safe-areas).
5. Capacitor-ready: relative paths, no network fonts (vendor woff2), safe-area insets, landscape lock assumptions, all motion killable by `prefers-reduced-motion`.
6. Palette = sa-p3.css tokens. Deviations in §1 are the ONLY ones.

## 1. Identity layer (both surfaces)
- **Type roles (EXPLICIT DEVIATION, showcase surfaces only):** display = **Fraunces** (variable; use two optical instances: soft display for plaques/titles at 600-ish weight, WONKY axis on for the wordmark moment) — replaces Space Grotesk *on these two surfaces only*; the app-wide trio elsewhere is untouched. Data = IBM Plex Mono (unchanged law: data values only, tabular-nums, U+2212). UI small text = Inter (unchanged). Vendor Fraunces woff2 (latin subset, 2 weights or the variable file if <300KB) into `vendor/fonts/`; @font-face scoped to these pages (NOT added to sa-p3.css).
- **Why the serif:** engraved-instrument-plaque voice — an observatory nameplate, not a sports HUD. It is the single loudest taste statement in the showcase; guard it by keeping ALL numbers mono.
- **Brass (deviation-by-promotion):** `--launch #E3C468` is promoted on these surfaces to the *etched-instrument* role: graduated ring scales, tick engravings, plaque rules, star-chart gridlines' major ticks. It may NEVER carry data or status.
- **Colour mapping (geometry, kills the teal era):** arc tube + glow gradient `#7ff4eb→#15b9ad` → **ember**: rest = `--accent #FF8A4D` with a cooler tail (`rgba(255,138,77,.35)`), during swing the tip runs white-hot `#FFF3E8` (documented light-carve-out, canvas/3D only). Target line cyan → neutral white-alpha (`--line-strong` grade, additive). Glass-plane grid `0x7ff4eb` → **`--plane #93A4F2`** (SYS-11 token). Low-point marker/bracket → **`--q-attack #4DE8D2`**. Delivery-arrow ground projection cyan → **`--q-path #6FC6FF`**; arrow body stays ink-neutral. fx.js cyan trail → ember-tail (`rgba(255,138,77,…)` ramp). Divot soil tones stay. Lighting: key light shifts from `0xdfeaf6` to a UV-violet key (`#C9CCFF`-class), fill deepens violet, hemisphere ground tint `#0A0714`; one warm ember rim/point light locked near the ball so the club edge catches fire-light. Depth fog: violet `#0B0817`, subtle.

## 2. SURFACE 1 — 3D geometry (`geometry.html` + `geo3d/`)
### 2.1 The Armillary (the plane, reborn)
Replace the holographic glass sheet's *dress* (geometry/maths untouched): the plane group gains a **graduated ring** — a thin torus/annulus lying IN the plane, brass-etched degree ticks every 5° with engraved numerals every 15° (texture or line-segments; brass `--launch`, additive, subtle), plus the existing grid demoted to `--plane` periwinkle at lower opacity. Where the plane meets the ground (the hinge line = swing direction), a small brass protractor arc shows the plane angle°, engraved. The plane READS as a scientific instrument ring, not a glass pane. Plane-angle slider live-rotates the ring; the etched readout at the hinge counts in mono via a DOM plaque anchored by `Vector3.project` (reuse lowpoint.js label pattern).
### 2.2 Star-chart ground
The instrument floor's radial ticks/rings restyle to a star chart: fine white-alpha graticule, brass major ticks, 5-6 faint constellation-style point clusters (procedural canvas texture — NO photo asset), vignette kept. Target line = one clean meridian.
### 2.3 Camera & gesture
- **Free orbit (new):** one-finger drag anywhere on the canvas orbits the rig — azimuth clamped to the useful arc (about −160°…−50°, between just-past-DTL and just-past-FACE), elevation clamped 6°…28°, with inertia (velocity decay) and **magnetic pole snap**: releasing within ~12° of a named pose glides into it (gravitational ease, 600ms). FACE/DTL buttons still work (tween through the same rig — they already do). Two-finger pinch = dolly within ±18% of pose dist. During drag: render loop active; on settle: back to render-on-demand. Reduced-motion: snap without glide. Keyboard: `[`/`]` step 15° azimuth; existing pose buttons remain the primary a11y path. Hit rule: drag begins anywhere not on a control (rail/chips excluded).
- **Idle telescope drift:** when idle >6s AND motion allowed AND page visible, a barely-perceptible azimuth drift (±1.5°, 14s period, sinus) — the observatory breathes. Any input kills it instantly. `visibilitychange` pauses. renderCount discipline: drift counts as active render window; verify battery story stays sane (drift renders at half rate — throttle to 30fps).
### 2.4 THE WOW — the Impact Replay (broadcast through the armillary)
On HIT (motion allowed): the existing timeline runs; the NEW camera choreography rides it:
- **Down-phase** (p 0.05→0.42): camera whips ~30° of azimuth toward the impact-side composition, tracking the club (GSAP tween keyed to timeline progress, not wall-time), while global timeScale ramps 1→0.3.
- **Impact window** (existing 0.42→0.58 slow-mo + facezoom FIX N): ramp deepens to the existing 0.04 slow-mo; **star-trail persistence** arms — 10-14 ghost club silhouettes accumulate along the path (instanced, additive, ember-tail alpha ramp, oldest dying) so the swing paints its own long-exposure.
- **The freeze** (existing p_impact freeze, unchanged machinery): add ONE ember bloom pulse on the contact marker (300ms), and a **Fraunces plaque** stamps the verdict beneath the strike chip — `PURE · 92%` (title in Fraunces, number mono) — typed-on like a lower-third (letters resolve in 240ms, no blur).
- **Release** (tap/Esc/8s, unchanged): camera completes its arc to a finish composition, timeScale 1, ghosts decay over 900ms, plaque holds until next input.
- **Reduced-motion:** exactly today's path (no whip, no ghosts, no ramp) + plaque renders instantly. Haptics unchanged (existing mapping). NOTHING about the freeze/hold/tap-to-continue logic is rewritten — the replay DECORATES the existing timeline+facezoom machinery via its progress callbacks.
### 2.5 Plaques & chips
PATH/ATTACK chips stay mono/param-coloured (law). Pose names, section labels, and the verdict word move to Fraunces plaques (DOM, anchored, tilt-to-camera ≤4° parallax under drag — cheap 2D transform, not 3D text). All plaque text AA on plate.

## 3. SURFACE 2 — front page (`home-mock.html`, transformed in place)
### 3.1 Structure — "your golf as a constellation over the range"
Keep the calibrated range-33 world + night grade as the LOWER third (horizon low): brand continuity. ABOVE the horizon, the sky becomes the interface — a procedural starfield (canvas, parallax-still) where **five module-stars** hang in a deliberate constellation (not a row): TEE OFF/Ball Flight low-left near the tee sightline, STRIKE WINDOW mid-left, OUTCOME mid-right (downrange), WHY?/Diagnose apex, ACADEMY high-right (the "pole star" with the XP ring around it). Thin constellation lines connect them in the pedagogy order (strike→flight→why→outcome→academy). Each star: a soft point + Fraunces plaque label + ONE mono live stat beneath (the sa.stat.* values — e.g. `162 m`, `87% FLUSH`, `LVL 7`). The ember ball rests on the tee below, the only hot element at rest (ember census ≤3: ball + one promoted stat + nothing else).
### 3.2 THE WOW — the Comet Broadcast (entry, once per session)
Cold entry (motion allowed, not yet played this session): the range wakes (existing floodlight beat compressed to ~0.9s) → **your actual last shot** (real params → `solveFlight` → mapped to a sky arc) crosses the constellation as an ember comet with a decaying star-trail, landing ON the module-star that owns its story (default: Ball Flight) — that star flares once; a **lower-third types itself** bottom-left in mono on a plate: `LAST SHOT · 162 m · DRAW · FLUSH 87%` with a Fraunces `Tonight at the range` eyebrow. Total ≤2.6s, any input skips, `sessionStorage` once-gate, no-stats fallback = the canonical demo shot labelled `THE HOUSE DRAW · try it →`. Reduced-motion: constellation + typed line render complete and static.
### 3.3 Interaction
- Tap a star (or its plaque — one hit area, ≥44px + generous halo): the ember ball LIFTS from the tee and travels the constellation line to that star (420ms, gravitational ease, comet-tail), star swallows to navigate (shared-element feel; plain fade under reduced-motion). Double-activation guard; mid-flight taps retarget honestly.
- Stars breathe (opacity 0.85↔1, desynced 7-9s periods); lines shimmer NEVER (static hairlines); IMU/pointer idle parallax ≤6px on the starfield only (sensor permission never requested — pointer/tilt fallback math only if already granted elsewhere; degrade silently).
- Keyboard/AT: stars are real `<a>` in pedagogy order, `.sa-focus` ring renders as a brass reticle around the star (still the codified double-ring underneath), one polite live region announces the comet line once. Help `?` stays top-right (existing contract). Wordmark top-left in Fraunces (the one WONKY-axis moment).
### 3.4 What dies
The 4-card rail. The card grammar (eyebrow/glyph/stat) migrates into star-plaques. Academy strip merges into the ACADEMY pole star's XP ring (SVG arc, `--celebrate` fill remains reward-only).

## 4. Assets & engineering
- Fonts: Fraunces variable woff2 (latin) → `vendor/fonts/`; per-page @font-face; `font-display: swap`.
- Starfields/star-chart: procedural canvas (seeded — reuse mulberry32 pattern) — crisp at DPR 2, zero asset weight. Nano Banana NOT needed for v1 (a nebula plate may be an iteration-pass complexify option; if used: generated 1600px, ≤300KB, UV-harmonized, documented).
- Ghost-club persistence: cloned/instanced meshes with shared material + per-ghost opacity, hard cap 14, pooled. No render-target accumulation (WKWebView safety).
- Performance budget: geometry page ≤22 draw calls resting (today ~20 + ring + chart), 60fps during replay on iPhone-class GPU, DPR cap 2 stands; front page: one canvas (starfield+comet) + DOM, no continuous rAF at rest (breathing = CSS).
- `www/` is NOT hand-synced (owner wires Codemagic; copy-web handles it) — root files just stay copy-web-compatible (they are).

## 5. Iteration protocol (the three passes, owner-mandated)
After the build verifies green, Fable (main session) runs three fine-toothed-comb passes over BOTH surfaces, each producing surgical fixes before "ok":
1. **Design pass** — hierarchy, plaque craft, brass discipline, type optical sizes, ember census, screenshot-graded at 932×430 + 812×375.
2. **Motion/interaction pass** — replay beat timings against the beat-sheet, orbit inertia feel, magnetic snap thresholds, comet travel, skip/interrupt honesty, reduced-motion parity.
3. **3D/complexify pass** — lighting depth, ring engraving legibility, fog tuning, ghost-trail alpha curve, one deliberate added flourish per surface IF it earns its place (candidates: nebula plate; plane-angle protractor micro-anim; constellation line draw-in on first paint).
A11y-lead review runs between build and pass 1; accesslint + keyboard walk re-verified after pass 3. Deploy to Vercel after pass 3 → final link.
