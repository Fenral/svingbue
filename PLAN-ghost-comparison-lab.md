# PLAN: Ghost comparison lab — pinned baseline, per-ghost identity, per-ghost control
Turn the flight overlay's ghost system from an opaque FIFO side-effect into a deliberate A/B surface: pin a baseline, identify each ghost, toggle ghosts individually — chips/badges only, zero new prose panels.

## Goal
In the Play-flight overlay the user can (1) pin the current shot as a white dashed BASELINE that never gets evicted, (2) see each grey ghost identified by a slot letter + the param that differs from the live shot at its landing dot, (3) tap compact legend chips (A/B/C) to show/hide each ghost individually, and (4) read the Δ-label against the PINNED baseline instead of only "whatever I just did". A one-time coach mark introduces the legend the first time a ghost appears. Everything is chips/badges/numbers — no new sentences on the persistent UI.

## Edge cases a weaker model would miss
- **Escape double-handling (found in exploration):** `coachMarks`'s keydown is document-CAPTURE (sa-firstrun.js:329) and `onFlightKey` is document-bubble (impact.html:3886) — same node, different phase visits, so `e.stopPropagation()` (step 1) is sufficient (`stopImmediatePropagation` not needed). Without step 1, Escape on the ghost coach bubble closes the whole overlay.
- **Pinned shot double-render:** the common flow is pin live → tweak → play, which makes the pinned shot ALSO become ghosts[0] via `noteFlight`. Step 4's guard prevents the duplicate capture. Consequence to accept: unpinning discards that shot entirely (it never entered the FIFO) — by design, do not "restore" it.
- **`buildGhostLayer` early-returns when `!FX.built || !FX.ghosts` (:3250)** — the `buildGhostLegend()` call at its end never runs before `ensureFX()`. That's fine: `openFlight` calls `ensureFX()` (:3868) before `fPlay` → `fBuild`.
- **Legend rebuild vs focus:** chips are destroyed on every `fBuild` (each new shot). The data-gid focus-restore in step 8 is mandatory, or a keyboard user's focus silently drops to `<body>` mid-session.
- **`revealData` fires repeatedly** — from every landing, every overlay resize (`renderStatic`, :3994) and every in-overlay speed change (:4036). `maybeGhostCoach` must be idempotent: the `ghostCoach` handle + `SA.seen` guard both required (markSeen only happens when the user finishes/skips the bubble).
- **Ghost pre-roll T0 (:3605):** with a baseline pinned but zero ghosts, the fade-in pre-roll must still run (step 9's `|| fBaseline`), else the baseline pops in at full opacity mid-flash.
- **Hidden ghosts stay in the camera fit** (step 9 passes ALL ghosts) — intentional: toggling a chip must not re-fit the camera and visually jump every other path. The shared-camera fit over live+ghosts is the load-bearing invariant; never fit only visible paths.
- **`dataset.op` drives the fade-in tween (:3615)** — the baseline group must set `grp.dataset.op = .55` or GSAP tweens it to `NaN`.
- **`fGhostPill` master toggle hides `FX.ghosts` wholesale (:4006)** — the baseline group lives inside `#fGhosts`, so "Ghosts off" hides the baseline too. Intentional (comparisons off = all comparisons); do not move the baseline to another layer.

## Accessibility requirements
- **Touch targets:** every `.fChip` is 36px tall with a `::after` extending ±4px vertically → 44px effective; 8px flex gap means adjacent expanded targets meet but never overlap. `.fChip` min width self-sizes >44px via padding+content.
- **State not by colour alone:** hidden ghost chip = `aria-pressed="false"` + strikethrough text + dimmed swatch; pinned = `aria-pressed="true"` + label text change (Pin → "216m"); baseline path differs from ghosts by dash AND colour AND the ⊙ badge glyph.
- **Roles/labels:** `#fGhostLegend` is `role="group" aria-label="Shot comparison"`. Pin button keeps a STABLE accessible name across states — `aria-label="Pin as baseline, 216 m"` (compute carry even while unpinned would-be text is just "Pin current shot as baseline"; once pinned, name becomes "Baseline pinned, 216 m") with `aria-pressed` carrying the toggle state alone — do not ALSO flip the name to the opposite verb ("Unpin") while `aria-pressed` flips; that double-signals SR users (ARIA APG: change state or name, not both). Ghost chips: accessible name includes the changed-param string, not just letter + carry — `"Ghost A, Face +4, 209 metres. Show or hide"` — because the in-scene landing badge shows WHICH parameter differs and the scene `<svg>` is decorative, so the chip name is the ONLY accessible carrier of that information (WCAG 1.1.1); `aria-pressed` still carries show/hide.
- **Live region:** `#fDelta` is already `aria-live="polite"` (:1126); the ⊙ prefix includes `<span class="sr-only">vs pinned baseline</span>` so SR users hear which reference is in play. No new live regions. Confirm `#fDeltaCause`'s existing writer sets `.innerHTML` (not `.textContent`) before relying on the `<span>` markup rendering as markup rather than literal text — if the current writer uses `textContent`, switch that write to `innerHTML` as part of this change. Add an `.sr-only` CSS rule to impact.html if the file does not already define one (standard clip-to-1px pattern) — otherwise the "vs pinned baseline" text renders visibly instead of being screen-reader-only.
- **Focus order:** legend sits in DOM after `fGhostClear`, so overlay tab order is close → replay → ghosts pill → clear → pin → A → B → C → scene/bottom chips. Focus-restore by `data-gid` after chip rebuild (step 8) covers the common case, but is incomplete on its own: when the focused chip's `data-gid` no longer exists after rebuild — the ghost was evicted from the FIFO by a 4th param change, or removed entirely by Clear — focus must NOT silently drop to `<body>`. Fall back to focusing the nearest remaining `.fChip-ghost`, or `#fPinBtn` if no ghost chips remain. Coach bubble manages its own trap and returns focus on close (sa-firstrun.js).
- **Contrast:** chip text `#9FB2C8` on `rgba(10,18,30,.7)` over the dark scene ≥ 7:1; white `#E8F0FB` higher; both pass AA 4.5:1. Off-state `var(--muted)` #9aabc6 on same plate ≈ 6.8:1 — passes. Keep the rgba plate behind all in-scene text.
- **Reduced motion:** no new animation is added; chips/legend render instantly; `renderStatic` path (reduce) shows baseline+ghosts at final opacity with `revealData(true)`. Coach bubble already honours `prefers-reduced-motion` internally.
- **Focus visible:** `.fChip:focus-visible` cyan 2px outline (matches the app pattern).
